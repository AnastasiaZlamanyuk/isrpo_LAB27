## Главные выводы

1. **Middleware — это мощный механизм.**  
   С их можно логировать запросы, проверять авторизацию, добавлять заголовки или прерывать цепочку до вызова основного обработчика.

2. **ASP.NET Core автоматически сериализует объекты C# в JSON.**  
   Достаточно вернуть `new { ... }` или `Results.Ok(...)`, и фреймворк сам установит `Content-Type: application/json`.

3. **Порядок регистрации middleware и маршрутов критичен.**  
   Если зарегистрировать `MapGet` до `app.Use(...)`, middleware не будет обрабатывать запросы к этому маршруту.

## Ответы на вопросы

### 1. Чем Minimal API отличается от MVC в ASP.NET Core?

| Характеристика | Minimal API | MVC |
|----------------|-------------|-----|
| Сложность | низкий порог входа | более сложный, много соглашений |
| Контроллеры | нет, вся логика в `Program.cs` | есть отдельные классы `Controller` |
| Подходит для | микросервисов, маленьких API | больших приложений с UI |
| Встроенные возможности | только самое нужное | полный набор (фильтры, модели, валидация) |

### 2. Что произойдёт, если поставить `app.MapGet(...)` до `app.Use(...)`?

Middleware **не будет выполняться** для этого маршрута, потому что запрос сразу попадёт в обработчик `MapGet` и не пойдёт дальше по цепочке.  
**Правило:** все `app.Use(...)` должны быть **перед** маршрутами.

### 3. Почему ASP.NET Core преобразует PascalCase в camelCase при сериализации JSON?

По умолчанию используется `System.Text.Json` с политикой именования `JsonNamingPolicy.CamelCase`.  
Это сделано для соответствия **стандартам JavaScript/TypeScript**, где принят `camelCase` (например, `firstName`, а не `FirstName`).

### 4. Что означают коды HTTP?

| Код | Название | Значение |
|-----|----------|-----------|
| 200 | OK | Всё хорошо, данные возвращены |
| 401 | Unauthorized | Требуется авторизация (ключ / логин) |
| 404 | Not Found | Ресурс не найден |

### 5. Чем `dotnet run` отличается от `dotnet watch run`?

| Команда | Поведение |
|---------|-----------|
| `dotnet run` | однократный запуск; при изменении кода нужно остановить и перезапустить вручную |
| `dotnet watch run` | **горячая перезагрузка** — автоматически перекомпилирует и перезапускает сервер при изменении файлов |
